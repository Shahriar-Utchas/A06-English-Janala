Frequently Asked Questions


1. The difference between var, let, and const
var is function-scoped and allows re-declaration, while let and const are block-scoped. let is hoisted but can be reassigned and does not allow re-declaration in the same block. const is also hoisted but must be initialized when declared and cannot be reassigned.

2. The difference between map(), forEach(), and filter()
map() transforms each element and returns a new array. forEach() is used for iteration and does not return anything. filter() creates a new array containing only elements that meet a condition.

3. Explain arrow functions and how they are different from regular functions?
Arrow functions provide a more concise syntax and do not have their own this context. They inherit this from the surrounding scope, unlike regular functions that have their own this context, which is determined by how they are invoked.

4. How JavaScript Promises work?
JavaScript Promises represent asynchronous operations and can be in one of three states: pending, resolved, or rejected. They allow you to handle asynchronous results with .then() for success or .catch() for errors, making asynchronous code more manageable.

5. How closures work in JavaScript?
A closure happens when an inner function remembers and can access variables from its outer (enclosing) function, even after the outer function has finished executing. This allows the inner function to keep using those variables, which is useful for creating private data and keeping things encapsulated.
